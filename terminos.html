Términos y condiciones

Última actualización: julio de 2026

Estos términos describen las condiciones de uso de la aplicación de escritorio TikLive TTS y de este sitio web. Al descargar o usar la app aceptas estas condiciones.

1. Software gratuito y de código abierto

TikLive TTS es un proyecto open source desarrollado de forma independiente. El uso de la aplicación es totalmente gratuito: no existen versiones de pago, suscripciones ni funciones bloqueadas detrás de un pago. El código fuente está disponible públicamente en GitHub, donde cualquiera puede revisarlo, proponer mejoras o reportar problemas.

2. Donaciones voluntarias

Las donaciones que aparecen en este sitio (Lightning Network y Bitcoin) son completamente voluntarias. Donar no otorga acceso a funciones adicionales, soporte prioritario ni ningún otro beneficio: es únicamente una forma libre de apoyar el desarrollo del proyecto. Las donaciones no son reembolsables y no constituyen un contrato de servicio.

3. Sin garantías

La aplicación se distribuye "tal cual" (as is), sin garantías de ningún tipo, expresas o implícitas. El desarrollador no se hace responsable de daños o pérdidas derivados del uso de la aplicación. Eres responsable de cumplir los términos de servicio de las plataformas que conectes (TikTok, Twitch, YouTube, Kick).

4. Independencia del proyecto

TikLive TTS es un proyecto independiente y no está afiliado, patrocinado ni respaldado por TikTok/ByteDance, Twitch, YouTube/Google, Kick ni OBS. Los nombres y marcas citados pertenecen a sus respectivos propietarios y se usan solo con fines descriptivos de compatibilidad.

5. Libertad del desarrollador

Al tratarse de un proyecto libre, gratuito y mantenido de forma voluntaria, el desarrollador se reserva el derecho de modificar, pausar o descontinuar la aplicación o partes de ella en cualquier momento, sin que ello genere obligaciones frente a los usuarios. El carácter open source garantiza que la comunidad siempre puede continuar el proyecto a partir del código publicado.

6. Contacto

Para dudas sobre estos términos puedes abrir un issue en GitHub o escribir en el Discord de la comunidad.
